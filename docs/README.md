## Documentation Standards

When adding new documentation:

1. **Use Clear Headings**: Organize content with hierarchical headings
2. **Include Examples**: Provide code examples and use cases
3. **Keep it Updated**: Review and update docs with code changes
4. **Link Related Topics**: Cross-reference related documentation
5. **Use Markdown**: Follow consistent Markdown formatting

## Contributing to Documentation

1. Check for existing documentation before creating new files
2. Follow the existing structure and style
3. Include practical examples
4. Test all commands and code snippets
5. Use screenshots where helpful
6. Keep language clear and concise
7. Update the table of contents when adding new documents

## Documentation Style Guide

- Use present tense ("Deploy the site" not "Deploys the site")
- Use active voice ("Run the script" not "The script is run")
- Be concise but complete
- Use code blocks for commands and code
- Use blockquotes for important notes
- Include prerequisites at the beginning of guides
